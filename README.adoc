= Defend, Contain, Comply
:toc: auto

A hands-on workshop demonstrating automated vulnerability response, policy-gated
patching, and compliance hardening using Red Hat Ansible Automation Platform.

== Workshop Modules

[cols="1,2,2"]
|===
| Module | Focus | Hands-on exercises

| *DEFEND -- Detect and Contain*
| Detect CVE-2024-38476 in Apache httpd, contain the vulnerable service, and harden the system posture -- all driven by Event-Driven Ansible responding to Splunk alerts.
| Write an EDA rulebook from a skeleton; configure EDA Controller manually (credential, project, rulebook activation).

| *CONTAIN -- Policy-Gated Remediation*
| Apply the RHSA-2024:5138 patch through an OPA policy-gated workflow with pre-patch validation, approval gates, post-patch verification, and compliance reporting.
| Write a pre-patch policy check playbook that queries OPA; build the Policy-Gated Patching workflow in the AAP UI.

| *COMPLY -- Audit, Harden, Deliver*
| Run a 13-control OPA compliance audit, apply CIS-aligned hardening, generate before/after evidence, then build a hardened container image, scan it, and push to a registry.
| Execute and observe the full compliance and delivery workflow.
|===

== Architecture

[cols="1,2"]
|===
| Host | Role

| `control`  | AAP Controller 2.6 + EDA Controller
| `central`  | Splunk (SIEM), OPA (policy engine), errata yum repo
| `rhel01`   | Target app server (Parasol Finance customer portal)
| `rhel02`   | Spare RHEL host
| `gitea`    | Git repository (containerized)
|===

== Setup

The automated setup provisions infrastructure and configures all services except EDA
and the Policy-Gated Patching workflow, which are hands-on exercises.

=== What the setup creates

* Central services: Splunk, OPA, errata repository
* App server: vulnerable httpd, firewalld, SELinux, Podman, container images
* AAP: credential, project (Gitea sync), inventory, 13 job templates, 2 workflows
  (Vulnerability Containment, Compliance and Hardening)

=== What attendees configure manually

* EDA Controller: credential, project, rulebook activation (Module 1)
* Policy-Gated Patching workflow in AAP (Module 2)

=== Opt-in automated setup

To include EDA and the patching workflow in automated setup:

[source,bash]
----
ansible-playbook setup-automation/main.yml --tags all,eda,patching_workflow
----
